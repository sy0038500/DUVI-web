# DUVI: 서울 행정동 도시활력지표 웹 시각화

> **“서울 426개 행정동의 도시활력을 데이터로 비교하고, 사용자의 목적에 따라 분야별 지표를 탐색하는 도시 데이터 시각화 플랫폼”**

---

## 1. 프로젝트 소개

DUVI(Dynamic Urban Vitality Index)는 정적인 시설 수 중심의 평가를 넘어, 인구 변화, 이동 흐름, 상권 활동, 주거위험 등 시간에 따라 변화하는 도시의 동적 활력을 진단하기 위해 구축한 복합 도시지표이다. 본 프로젝트는 상권, 인구, 주택시장 및 주거위험, 생활시설, 교통, SDOT 유동인구, 에너지, 생활인구 등 다양한 도시 데이터를 행정동 단위로 결합하고, 이를 웹 지도와 랭킹 형태로 시각화한다.

기존의 단일 지표 중심 도시 분석은 특정 지역의 활력을 종합적으로 설명하기 어렵다는 한계가 있다. 이에 본 프로젝트는 여러 분야의 도시 데이터를 통합하여 행정동별 도시활력을 비교할 수 있도록 하였고, 동시에 사용자가 관심 있는 분야를 개별적으로 탐색할 수 있는 세부탐색 기능을 제공한다.

웹사이트에서는 2022년부터 2026년까지의 연도별 DUVI 결과를 확인할 수 있으며, 종합 DUVI뿐 아니라 상권, 인구, 주택위험, 생활시설, 교통, SDOT 유동인구, 에너지, 생활인구 등 분야별 세부 지표도 별도 지도에서 탐색할 수 있다. 또한 안전지표는 원자료의 공간 단위에 맞추어 자치구 단위 보조지표로 제공한다.

## 3. 주요 기능

### 3.1 종합 DUVI 지도

- 서울시 426개 행정동별 DUVI 점수 시각화
- 2022년, 2023년, 2024년, 2025년, 2026년 선택 가능
- DUVI 최종 점수를 7등급으로 분류하여 지도 색상으로 표현
- 행정동 클릭 시 DUVI 점수, 등급, 영역별 점수 확인 가능
- 연도별 DUVI 상위 10개 행정동 랭킹 제공

### 3.2 분야별 세부탐색

종합 DUVI는 여러 영역을 동일 가중 방식으로 결합한 결과이다. 그러나 사용자의 목적에 따라 중요하게 보는 기준은 달라질 수 있다. 예를 들어 이주를 고려하는 사용자는 생활시설과 주거위험을, 상권 분석 사용자는 상권과 유동인구를, 교통 접근성을 보는 사용자는 버스·지하철 지표를 우선적으로 확인할 수 있다.

이에 따라 세부탐색 페이지에서는 다음 분야별 지표를 개별 지도와 순위로 확인할 수 있다.

- 종합 DUVI
- 상권
- 인구
- 주택시장 및 주거위험
- 생활시설
- 버스교통
- 지하철교통
- SDOT 유동인구
- 에너지
- 생활인구
- 안전

### 3.3 지도 색상 체계

- 종합 DUVI 지도는 최종 7등급 색상표를 사용한다.
- 분야별 세부탐색 지도는 각 분야의 대표 색상을 사용하며, 점수가 높을수록 진하게, 낮을수록 연하게 표현한다.
- SDOT 센서 자료나 공동주택 에너지 자료가 없는 행정동은 실제 값이 0이라는 뜻이 아니라 해당 원자료의 관측값이 없는 지역이므로 회색의 `미산정/결측`으로 표시한다.
- 안전지표는 자치구 단위 자료이므로 행정동 지도가 아닌 자치구 경계 지도에서 별도로 표시한다.

### 3.4 행정동 및 자치구 상세 패널

- 행정동 클릭 시 DUVI 점수, 등급, 사용 영역 수, 사용 변수 수 확인
- 영역별 점수 막대 그래프 제공
- 세부탐색에서 선택한 분야별 점수와 상위·하위 지역 랭킹 제공
- 안전지표 선택 시 자치구 단위 점수와 자치구 랭킹 제공
- SDOT 및 에너지 자료가 없는 행정동은 0점이 아닌 `미산정/결측`으로 표시

## 5. 사용자 작업 흐름(User Task Flow)

본 서비스는 사용자가 종합 도시활력 수준을 먼저 파악하고, 이후 자신의 목적에 맞는 세부 지표를 탐색할 수 있도록 설계하였다.

> **서비스 접속 → 연도 선택 → 종합 DUVI 지도 확인 → 행정동 클릭 → 상세 점수 확인 → 세부탐색 이동 → 분야 선택 → 연도/기간 선택 → 분야별 지도 및 랭킹 확인**

이 흐름을 통해 사용자는 단순히 최종 점수만 보는 것이 아니라, 특정 행정동이 왜 높거나 낮게 평가되었는지 영역별로 확인할 수 있다.

## 6. AI 활용 및 개발 워크플로우

본 프로젝트는 GPT 계열 생성형 AI를 데이터 처리, 지표 설계, 웹 구현, 오류 해결, 문서화 전 과정에 보조적으로 활용하였다. AI는 최종 판단을 대신하는 도구가 아니라, 복잡한 공공데이터 처리 절차를 빠르게 구조화하고 반복적인 코드 작성과 오류 검토를 지원하는 개발 보조 도구로 활용하였다.

- **데이터 전처리 코드 생성 및 검수 보조**: CSV 파일의 인코딩 처리, 행정동 코드 통일, 행정동 단위 집계, 연도별 패널 생성, 결측 및 제외 행 검수 과정에서 GPT를 활용하였다.
- **공간 단위 정리 및 매칭 로직 설계**: 지하철역, 버스정류장, SDOT 센서, 공동주택 단지 등 서로 다른 공간 단위의 데이터를 행정동 단위로 결합하기 위한 코드 구조와 검수 로직을 설계하였다.
- **지표 산정 방식 구조화**: 정방향 지표와 역방향 지표를 구분하고, 0~100점 정규화, 영역별 평균, 최종 DUVI 산정, 7등급화 기준을 정리하는 데 AI를 활용하였다.
- **결측 처리 원칙 정리**: SDOT 센서 자료나 공동주택 에너지 자료가 없는 행정동을 실제 0점으로 해석하지 않고 `미산정/결측`으로 표시하는 원칙을 정리하였다.
- **웹 시각화 구현 보조**: React, Vite, Leaflet, PapaParse를 활용한 CSV·GeoJSON 로딩, 행정동 지도 시각화, 분야별 세부탐색 페이지, 자치구 단위 안전지도 구현 과정에서 GPT를 활용하였다.
- **UX/UI 설계 보조**: 종합지도와 세부탐색의 정보 구조, 분야별 색상 체계, 범례, 상세 패널, 랭킹 패널, 네비게이션 구조를 설계하는 과정에서 AI를 활용하였다.
- **배포 및 트러블슈팅**: GitHub Pages 배포 과정에서 발생한 경로, 빌드, 캐시, React 런타임 오류를 해결하는 데 AI를 활용하였다.
- **문서화 및 제출자료 정리**: README, 기획서 문구, 데이터 한계, 향후 개선 방향, AI 활용 내역 정리에 GPT를 활용하였다.

개발 흐름은 다음과 같다.

> **원자료 수집 및 전처리 → 행정동·자치구 단위 집계 → 연도별 DUVI 패널 생성 → 0~100점화 및 7등급화 → 웹 시각화용 CSV/GeoJSON 정리 → React/Vite 구현 → Leaflet 지도 연동 → GitHub Pages 배포**

## 7. 핵심 구현 기능

1. **행정동 단위 데이터 결합**
   - 상권, 인구, 주택위험, 생활시설, 교통, SDOT, 에너지, 생활인구 데이터를 행정동 코드 기준으로 통합하였다.

2. **연도별 DUVI 패널 구축**
   - 2022년부터 2026년까지 서울시 426개 행정동 × 5개년 구조의 패널 데이터를 구축하였다.

3. **0~100점 정규화 및 역방향 처리**
   - 단위가 다른 지표를 비교하기 위해 최소-최대 정규화를 적용하였다.
   - 전세가율, 건축물 연식, 에너지 비용 등 위험 또는 부담이 큰 변수는 역방향으로 점수화하였다.

4. **최종 DUVI 7등급화**
   - 최종 DUVI 점수를 연도별 7분위로 나누어 지도화와 해석이 가능하도록 하였다.

5. **Leaflet 기반 행정동 지도 시각화**
   - 서울 행정동 GeoJSON과 DUVI 결과 CSV를 결합하여 행정동별 등급 지도를 구현하였다.

6. **분야별 세부탐색 페이지**
   - 종합 DUVI 외에도 상권, 인구, 주택위험, 생활시설, 버스, 지하철, SDOT, 에너지, 생활인구, 안전 지표를 개별 지도와 랭킹으로 확인할 수 있도록 구현하였다.

7. **행정동 상세 정보 패널**
   - 선택한 행정동의 최종 DUVI 점수, 등급, 영역별 점수, 사용 변수 수를 제공하였다.

8. **사용자 중심 네비게이션**
   - 헤더의 종합지도/세부탐색 버튼과 사이드 스텝 네비게이션을 통해 사용자가 원하는 섹션으로 빠르게 이동할 수 있도록 설계하였다.

## 9. DUVI 산정 방식

### 9.1 지표 결합 단위

DUVI 산정에 활용되는 대부분의 지표는 서울시 행정동 코드(`admin_dong_code`)를 기준으로 결합하였다. 역, 센서, 공동주택 단지 등 서로 다른 공간 단위의 원자료는 행정동 단위로 재집계하였다. 다만 안전지표는 원자료가 자치구 단위이므로 행정동 DUVI 산정에는 직접 포함하지 않고, 세부탐색 화면에서 자치구 단위 보조지표로 별도 제공하였다.

### 9.2 연도별 산정 원칙

DUVI는 2022년부터 2026년까지 산정하였다.

- 2022~2024년: 해당 연도 데이터가 있는 지표를 중심으로 산정하고, 단년 또는 최신 기반 고정지표는 필요 시 보조지표로 포함하였다.
- 2025~2026년: 해당 연도 데이터가 있는 지표를 우선 사용하고, 해당 연도 데이터가 없는 경우 가장 최신 연도 또는 최신 기간 지표를 대체로 사용하였다.

예를 들어 상권은 2022~2025년 자료를 사용하며, 2026년에는 2025년 상권 최신값을 대체 사용하였다. 인구는 2022~2024년 자료를 사용하며, 2025년과 2026년에는 2024년 인구 최신값을 대체 사용하였다.

### 9.3 정규화 방식

각 지표는 단위와 규모가 서로 다르기 때문에 직접 합산하지 않고, 연도별 행정동 기준 최소-최대 정규화를 통해 0~100점으로 변환하였다.

값이 높을수록 도시활력이 높은 변수는 정방향으로 점수화하였다. 반대로 전세가율, 건축물 연식, 에너지 비용처럼 값이 높을수록 위험 또는 부담이 큰 변수는 역방향으로 점수화하였다.

### 9.4 영역별 점수

개별 변수별 0~100점화 결과를 바탕으로 같은 영역 내 변수들의 평균을 계산하여 영역별 점수를 산정하였다.

사용한 영역은 다음과 같다.

- 상권
- 인구
- 주택시장 및 주거위험
- 생활시설
- 버스교통
- 지하철교통
- SDOT 유동인구
- 에너지
- 생활인구

### 9.5 최종 DUVI 점수

최종 DUVI 점수는 각 영역별 점수의 평균으로 산정하였다. 결측값은 0으로 대체하지 않고, 해당 행정동·연도에서 사용 가능한 변수와 영역만 평균에 반영하였다. 웹 시각화에서도 SDOT 센서 자료나 공동주택 에너지 자료가 없는 행정동은 0점으로 해석하지 않고, `미산정/결측`으로 표시하였다.

### 9.6 DUVI 7등급화

최종 DUVI 점수는 연도별로 7분위 등급화하였다.

| 등급 | 의미 |
|---|---|
| 1등급 | 매우 높음 |
| 2등급 | 높음 |
| 3등급 | 다소 높음 |
| 4등급 | 보통 |
| 5등급 | 다소 낮음 |
| 6등급 | 낮음 |
| 7등급 | 매우 낮음 |

1등급일수록 해당 연도 내 상대적으로 도시활력 수준이 높은 행정동이며, 7등급일수록 상대적으로 낮은 행정동을 의미한다.

## 13. 데이터 한계 및 해석 유의사항

본 프로젝트는 다양한 공공데이터를 행정동 단위로 통합하여 DUVI를 산정하였으나, 다음과 같은 한계가 있다.

### 13.1 기준시점 차이

모든 데이터가 동일한 기간을 가지는 것은 아니다. 상권은 2022~2025년, 인구는 2022~2024년, 버스·지하철은 2026년 3~4월, SDOT은 2026년 4~5월, 생활인구는 2026년 1~4월 자료를 활용하였다. 따라서 2025~2026년 DUVI는 해당 연도 자료가 없는 일부 지표에 대해 최신 연도 또는 최신 기간 자료를 대체 사용하였다.

### 13.2 공간 단위 차이

원자료의 공간 단위가 행정동, 주소, 좌표, 지점, 단지, 자치구 등으로 서로 다르다. 버스정류장, 지하철역, SDOT 센서, 공동주택 단지 등 점 단위 자료는 좌표 기반 공간조인을 통해 행정동 코드를 부여하였다. 이 과정에서 경계부에 위치한 시설이나 지점은 실제 생활권과 행정동 경계가 완전히 일치하지 않을 수 있다.

### 13.3 결측 및 미산정 지역

SDOT 센서가 설치되지 않았거나 공동주택 에너지 자료가 없는 행정동은 해당 영역을 실제 0점으로 해석하지 않고 `미산정/결측`으로 표시하였다. 이는 실제 유동인구나 에너지 사용량이 0이라는 의미가 아니라, 해당 원자료의 관측값 또는 수집 대상이 없는 지역임을 의미한다.

### 13.4 안전지표의 공간 단위

안전지표는 원자료가 자치구 단위이므로 행정동 DUVI 산정에는 직접 포함하지 않았다. 대신 세부탐색 화면에서 자치구 경계 기반 보조지표로 제공하였다. 따라서 안전지표는 행정동별 DUVI 점수와 직접 비교하기보다 자치구 단위의 참고 지표로 해석해야 한다.

### 13.5 상대평가 기반 점수

DUVI의 0~100점화와 7등급화는 서울시 행정동 내부의 상대적 비교를 위한 것이다. 따라서 DUVI 점수는 절대적인 도시활력 수준이라기보다, 해당 연도 서울시 행정동 간 상대적 활력 수준을 나타낸다. 연도별 정규화 기준이 달라질 수 있으므로 장기 시계열 해석에는 주의가 필요하다.

### 13.6 현재 분석 범위

본 프로젝트는 서울시 426개 행정동을 대상으로 한 1차 PoC이다. 서울 외곽 및 인접 지자체와 연결되는 생활권 흐름은 충분히 반영하지 못한다. 향후 수도권 및 전국 단위로 확장하면 경계 지역의 이동 흐름과 생활권 특성을 더 정밀하게 반영할 수 있다.

## 14. 회고 및 인사이트

- **데이터 통합의 어려움**: 각 원자료의 공간 단위가 행정동, 역, 센서, 공동주택 단지 등으로 달라 이를 하나의 행정동 단위로 재집계하는 과정이 중요했다.
- **연도별 비교의 한계와 보완**: 모든 데이터가 동일한 연도 범위를 갖고 있지 않았기 때문에, 2022~2024년은 해당 연도 데이터 중심으로, 2025~2026년은 최신 자료 대체 원칙을 적용하였다.
- **종합지표와 세부지표의 병행 필요성**: DUVI는 여러 영역을 동일 가중으로 결합한 종합지표이지만, 사용자의 목적에 따라 중요한 기준은 달라질 수 있다. 이에 따라 세부탐색 페이지를 별도로 구성하여 분야별 지표를 확인할 수 있도록 하였다.
- **시각화 UX의 중요성**: 지도 색상, 범례, 행정동 상세 패널, 분야별 대표색 등을 통해 사용자가 점수와 등급을 직관적으로 이해할 수 있도록 설계하였다.
- **기술적 성장**: React와 Leaflet을 활용하여 CSV 및 GeoJSON 기반 지도 시각화를 구현하면서, 공공데이터 기반 웹 시각화의 전체 흐름을 경험하였다.
